# 연구 목적 대비 결과 냉철한 평가 (2026-04-06)

**목적**: 원래 연구 가설이 실험으로 의미 있게 검증되었는가? 논문으로서 가치가 있는가?

---

## 1. 원래 가설 vs 실제 결과 — 하나씩 대조

### RQ1: 주파수 vs 시간 도메인 특징

| 항목 | 원래 가설 | 실제 결과 | 판정 |
|------|----------|----------|------|
| H1.1 SpecCNN > Rule (주기 데이터) | AUC-PR 유의 차이 | SKAB: 0.446 vs 0.367 (p=0.0006) | **부분 지지** |
| H1.2 Rule > SpecCNN (스파이크) | Event Recall 비교 | 미검증 (이상 유형별 층화 분석 미실행) | **미검증** |
| H1.3 Hybrid ≥10% 향상 | 앙상블 효과 | Hybrid 0.375 < 단독 최고 0.446 — 오히려 하락 | **기각** |
| H1.4 데이터 특성으로 예측 | 예측 모델 | 미실행 | **미검증** |

**냉정한 평가**: RQ1은 4개 가설 중 1개만 부분 지지. SpecCNN이 SKAB에서 1위인 건 맞지만,
**Friedman p=0.194로 통계적으로 유의하지 않음**. SMD에서 3위로 떨어짐.
"주파수 도메인이 우수하다"는 주장은 **데이터 의존적이며, 통계적으로 견고하지 않다**.

**가장 큰 문제**: 이상 유형별 층화 분석(spike/step/drift)을 안 했음 — RQ1의 핵심인데 빠져있음.

---

### RQ2: 앙상블 방법

| 항목 | 원래 가설 | 실제 결과 | 판정 |
|------|----------|----------|------|
| H2.1 Linear ECE < 0.05 | 캘리브레이션 비교 | **기존 ECE는 leakage**: 0.01 (무효). 수정 후 0.19 | **기각** |
| H2.2 Max > Product (고불균형) | 비용 최적화 | 0.324-0.344 범위, 차이 무의미 | **기각 (negative)** |
| H2.3 Learned ≥5% 향상 | 학습 앙상블 | N=5로만 테스트, 차이 없음 | **기각** |

**냉정한 평가**: RQ2는 **완전한 negative result**. 앙상블 방법 간 차이가 없다.
그 자체로 보고할 가치가 있으나, 현재 논문 프레임에서 부각되지 않고 있음.
**ECE < 0.05 달성 주장은 calibration leakage 때문에 완전히 무효화됨.**

### RQ3: Point vs Event 메트릭 상관

| 항목 | 원래 가설 | 실제 결과 | 판정 |
|------|----------|----------|------|
| H3.1 ρ = 0.5~0.8 | 중간 상관 | SKAB ρ=0.82-0.85, SMD ρ=0.09 | **부분 지지** |
| H3.2 긴 이상 > 높은 상관 | 길이별 층화 | 짧은 시계열(SKAB)에서 높고 긴 시계열(SMD)에서 낮음 | **반대 방향** |

**냉정한 평가**: 흥미로운 발견이지만 **분석 깊이 부족**.
"짧은 시계열에서 상관이 높다"는 것은 **이상 길이 효과가 아니라 시계열 길이 효과**일 수 있음.
confound 분석이 없어서 인과적 해석 불가.

### RQ4: 비용 민감도

| 항목 | 원래 가설 | 실제 결과 | 판정 |
|------|----------|----------|------|
| H4.1 ratio ≈ 0.5 × imbalance | 선형 관계 | FN:FP=5:1 최적, 이후 포화 | **부분 지지** (비선형) |
| H4.2 20:1에서 포화 | 포화점 | 10:1에서 이미 포화 | **근접** |
| H4.3 낮은 SNR → 보수적 | SNR 효과 | SKAB에서만 테스트 | **미검증** (1개 데이터셋) |
| H4.4 ECE<0.05 → 강한 상관 | 캘리브레이션 효과 | **ECE<0.05 달성 못함** (leakage 수정 후) | **전제 붕괴** |

**냉정한 평가**: 5:1 최적이라는 발견은 실무적으로 유용하나,
**SKAB 한 데이터셋에서만 fully tested**. 일반화 주장이 약함.

### C1 (주력): Feature Attribution → LLM 설명

| 항목 | 원래 기대 | 실제 결과 | 판정 |
|------|----------|----------|------|
| 다중 센서 효과 | AUC-PR ≥10% 향상 | KW +24%(AIHub), +98%(SMD) | **강하게 지지** |
| 단일 센서 효과 0 | 대조군 | SKAB 효과 0% | **확인** |
| 센서 이름 > z-score | 메커니즘 | names_only 91% vs zscore_only 0% | **강하게 지지** |
| Top-3 최적 | 정보량 | sensor_id 4.65(top3) > 3.32(full) | **지지** |
| Spearman ρ ≈ 1.0 | 센서 수 상관 | **ρ ≈ 0.8** (수정됨) | **과장→수정** |
| FN에서도 효과 | TP-only bias 검증 | **FN sensor=90% vs TP 50%** | **예상 밖의 강한 결과** |
| 방향 정확도 | increase/decrease | **모든 조건 0%** | **실패** |

**냉정한 평가**: C1은 가장 강한 기여. 하지만 근본적 문제가 있음:

1. **"LLM에 정보를 더 주면 답이 좋아진다"는 당연한 결론 아닌가?**
   → 반론: "센서 이름만 주면 되고 z-score는 필요 없다"는 **비자명한** 발견
   → 반론: "Top-3이 전체보다 낫다(정보 과부하)"는 **비자명한** 발견
   → 하지만 이것이 **논문 한 편의 기여로 충분한가**는 여전히 의문

2. **Keyword faithfulness가 주 metric인 문제**
   → LLM-judge에서는 **전체 차이 없음** (p=0.922) — sensor_id 차원에서만 개선
   → 새로 만든 strict metric에서 방향 정확도 **0%** — LLM이 방향을 맞추지 못함
   → **"faithfulness가 높아졌다"는 주장의 실질적 의미가 의심됨**

3. **N이 작음**: SMD 10 TP + 10 FN = 20. AIHub 기존 34. 통계 검정력 불확실.

## 2. 논문으로서의 의미 — 솔직한 판단

### 의미 있는 것 (논문에 쓸 수 있는 것)

1. **C1: "센서 이름 > z-score" + "Top-3 최적"**
   - 이것은 진짜 비자명한 발견임
   - 메커니즘: LLM은 수치보다 의미론적 레이블에 의존한다
   - **단, N을 늘리고 human evaluation이 있어야 설득력이 있음**

2. **C2: Rank reversal 3개 데이터셋 재확인**
   - 원래 알려진 것이지만, **SpecCNN(주파수) vs kNN(값밀도) 역전**은 구체적 기여
   - SMD에서 Friedman p<0.000001, 모든 비교 유의 → 통계적으로 견고

3. **Calibration leakage 발견 및 수정**
   - ECE 16.5배 상승 — 이 자체가 방법론적 기여
   - "temporal split이 필수"라는 실무 가이드라인

4. **FN에서 feature attribution이 더 효과적**
   - 예상 밖 발견: 탐지기가 놓친 이상에서 sensor ID 90%
   - **논문의 가장 신선한 기여가 될 수 있음**

### 의미 없거나 약한 것

1. **"주파수 도메인이 우수하다" (RQ1)** — SKAB에서만 1위이고 Friedman 유의하지 않음
2. **"앙상블 방법 비교" (RQ2)** — negative result, 차이 없음
3. **"Threshold > Detector" (C3)** — 이미 알려진 것의 수치 확인
4. **"ECE가 낮다" (기존 주장)** — calibration leakage로 무효화됨
5. **"ρ=1.0 완벽 상관"** — 계산 오류, 실제 ρ≈0.8

### 절대 성능 문제 (리뷰어가 반드시 지적)

| 데이터셋 | 최고 AUC-PR | Random Baseline | 실질 Lift |
|----------|:----------:|:---------------:|:---------:|
| SKAB | 0.446 | ~0.35 | **+0.10 (28%)** |
| SMD | 0.390 | ~0.10 | +0.29 (good) |
| AIHub | 0.661 | ~0.54 | **+0.12 (22%)** |

- SKAB/AIHub에서 random 대비 lift가 매우 작음
- **"성능이 이렇게 낮은 탐지기들을 비교해서 의미가 있나?"** → 예상 리뷰어 질문

## 4. 논문 프레임 재설계 제안

현재 프레임: "Feature Attribution이 LLM 설명을 좋게 한다"
→ **문제**: 너무 당연하게 들림

### 제안 프레임 A: "LLM이 뭘 보는가" (메커니즘 중심)

**제목**: "What Do LLMs Actually Use? Sensor Names Beat Numerical Features
for Anomaly Explanation in Multi-Sensor Time Series"

**스토리**:
1. LLM에 feature attribution을 줄 때, 수치(z-score)는 의미 없고 **이름이 핵심**
2. 정보를 더 주면 오히려 나빠짐 (Top-3 > Full → 정보 과부하)
3. 탐지기가 놓친 이상(FN)에서도 attribution이 효과적 → 실무 가치
4. 이 효과는 센서 수 8개 이상에서만 발현

**장점**: "당연한 것" 프레임이 아니라 "LLM의 행동 분석" 프레임
**약점**: 여전히 human evaluation 필요

### 제안 프레임 B: "Calibration Matters" (방법론 중심)

**제목**: "Temporal Calibration Splits and Their Impact on Anomaly Detection
Evaluation: A Multi-Dataset Study"

**스토리**:
1. Calibration data leakage가 ECE를 16.5배 왜곡시킴
2. 올바른 temporal split 적용 시 "잘 캘리브레이션됨" 주장 붕괴
3. 탐지기 간 차이는 SKAB에서 통계적으로 유의하지 않음 (power 부족)
4. 실무 가이드라인: temporal split + power analysis 필수

**장점**: 방법론적 기여가 명확, 재현 가능
**약점**: novelty가 더 낮음

### 제안 프레임 C: 현재 프레임 강화 (가장 현실적)

**제목**: 현재 유지하되 3가지 보강
1. Human evaluation 추가 (N=30, 2명)
2. "센서 이름 vs z-score" 메커니즘을 전면에
3. FN에서의 효과를 새로운 발견으로 부각

## 5. 결론: 이 연구는 의미 있는가?

### 의미 있는 부분 (YES)
- **센서 이름 > z-score**: 비자명하고 실증적
- **Top-3 > Full ranking**: 정보 과부하 효과 실증
- **FN에서 attribution 효과**: 예상 밖 발견, 실무 가치 높음
- **Calibration leakage 정량화**: 방법론적 경고

### 의미 부족한 부분 (NO, or NOT YET)
- **RQ1 (주파수 우위)**: 통계적으로 유의하지 않음
- **RQ2 (앙상블)**: negative result
- **방향 정확도**: 0% — LLM이 실제로 데이터를 "이해"하지 못함
- **Human evaluation 없음**: XAI 논문의 치명적 약점
- **절대 성능 낮음**: SKAB/AIHub에서 random+10% 수준

### 최종 판정

**국내 학회 (KSC, KIISE)**: ✅ 현재 상태로 충분
**산업 저널 (IEEE Access, Sensors)**: ⚠️ human eval + 샘플 확대 필요
**Mid-tier 국제 (CIKM, PAKDD)**: ⚠️ 프레임 A + human eval + 메커니즘 분석
**Top-tier (NeurIPS, ICML)**: ❌ novelty 근본적 부족
